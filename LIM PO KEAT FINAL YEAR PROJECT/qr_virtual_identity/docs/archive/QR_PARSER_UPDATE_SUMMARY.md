# QR Parser Compatibility Update Summary

## 🎯 Objective

Updated all QR parsing and processing modules to support the **new dynamic JSON QR format** generated by the Profile Page while maintaining backward compatibility with the old `USER:<user_id>` format.

## 📋 Problem Statement

The mobile scanner was returning "Invalid QR Code" when scanning the new dynamic QR format from the Profile Page because the parser only supported the old colon-separated format.

### Old Format (Still Supported)

```
USER:<user_id>
```

### New Dynamic Format (Now Supported)

```json
{
  "uid": "abc123xyz789",
  "email": "user@example.com",
  "name": "John Doe",
  "role": "user",
  "ts": 1700227200000
}
```

## ✅ Changes Made

### 1. **lib/utils/qr_parser.dart** - Core Parser Updates

#### Added Import

```dart
import 'dart:convert';
```

#### Updated QrParseResult Model

Added new fields to support JSON QR data:

- `email` (String?) - User email from JSON QR
- `name` (String?) - User full name from JSON QR
- `role` (String?) - User role from JSON QR
- `timestamp` (int?) - Timestamp for dynamic QR validation

#### Updated Factory Constructor

```dart
factory QrParseResult.user(
  String raw,
  String userId, {
  String? email,
  String? name,
  String? role,
  int? timestamp,
})
```

#### Updated Parse Method

Added JSON detection and parsing **before** checking for colon-separated formats:

```dart
// 0. Check for JSON format first (new dynamic user QR from profile page)
if (cleanData.startsWith('{') && cleanData.endsWith('}')) {
  try {
    final jsonData = jsonDecode(cleanData) as Map<String, dynamic>;

    if (jsonData.containsKey('uid')) {
      final uid = jsonData['uid'] as String?;
      return QrParseResult.user(
        qrData,
        uid,
        email: jsonData['email'] as String?,
        name: jsonData['name'] as String?,
        role: jsonData['role'] as String?,
        timestamp: jsonData['ts'] as int?,
      );
    }
  } catch (e) {
    // Continue to other format checks
  }
}
```

#### Updated Type Detection

```dart
static QrType detectType(String qrData) {
  // Check for JSON format
  if (cleanData.startsWith('{') && cleanData.endsWith('}')) {
    try {
      final jsonData = jsonDecode(cleanData) as Map<String, dynamic>;
      if (jsonData.containsKey('uid')) {
        return QrType.user;
      }
    } catch (e) { }
  }

  // Check other formats...
}
```

## 🔄 Backward Compatibility

All existing QR formats continue to work:

| Format                              | Status     | Example                                           |
| ----------------------------------- | ---------- | ------------------------------------------------- |
| `TICKET:event_id:user_id:timestamp` | ✅ Working | `TICKET:EVT001:user123:1700227200000`             |
| `USER:<user_id>`                    | ✅ Working | `USER:abc123xyz789`                               |
| `ITEM:<item_id>`                    | ✅ Working | `ITEM:BOOK001`                                    |
| `SCANPOINT:<id>`                    | ✅ Working | `SCANPOINT:SP001`                                 |
| JSON User QR                        | ✅ **NEW** | `{"uid":"abc123","email":"user@example.com",...}` |
